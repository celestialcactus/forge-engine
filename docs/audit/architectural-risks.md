# ForgeEngine Architectural Risks

Priority scale: **P0** blocks any untrusted execution; **P1** blocks a credible baseline; **P2** should be resolved before feature expansion; **P3** is roadmap cleanup.

## Risk register

| Priority | Risk | Concrete evidence | Impact | Recommendation |
|---|---|---|---|---|
| P0 | Arbitrary host command execution | `src/tools/bash.ts` passes model-provided text to `child_process.exec()` with the host cwd. Its Docker claim is a comment. `ToolRegistry` never calls `ConstraintEngine.validateCommand()`. | Host compromise, secret access, destructive commands, network exfiltration. | Do not expose `bashTool` to an LLM until execution is placed in a separately enforced sandbox with resource, filesystem, process, and network controls. |
| P0 | Uncontained filesystem access | `read-file.ts` and `write-file.ts` use `path.resolve(cwd, input.filePath)` without containment. Registry constraint validation is empty. | Read/write anywhere allowed to the process, including credentials and unrelated repositories. | Centralize canonical path validation, prevent symlink/reparse-point escape, separate read/write roots, and test platform-specific bypasses before execution. |
| P0 | Safety does not cover model egress | `AgentDispatcher` sends task/system/context to `generateText()` directly. DLP is only post-tool-output; egress only checks `web` tool URLs. | Secrets or proprietary content can leave sovereign/managed boundaries despite policy configuration. | Put provider construction and every outbound request behind a single mandatory policy gateway; apply DLP before egress and produce tamper-evident decisions. |
| P0 | MCP is an unrestricted subprocess boundary | `connectMcpServer()` accepts command/args and merges all of `process.env`; mapped tools default to `execute`. | Supply-chain execution, credential leakage, unconstrained filesystem/network access. | Require admin-approved server manifests, minimal environment, executable allowlist/digest, capability declarations, isolation, lifecycle limits, and audit. |
| P1 | Clean installation is not reproducible | `npm ci` fails because `ai@4.3.19` conflicts with `ai-sdk-ollama@3.8.8` requiring AI SDK 6. | Users/CI cannot create the claimed baseline without weakening peer checks; runtime compatibility is unverified. | Choose and pin a compatible AI SDK/provider generation, regenerate the lockfile deliberately, and test clean installs on supported platforms. |
| P1 | No runnable engine or CLI | `src/engine.ts` is absent; `src/cli/index.ts` has no top-level invocation; `ForgeCLI.start()` only renders timers. | No end-to-end product behavior can be verified. | Define a minimal composition root and one deterministic mocked workflow only after boundary ADRs; keep CLI claims narrow. |
| P1 | Permission and limits are policy theater | Config declares approvals, runtime minutes, iterations, and tool-call limits, but no executor consumes most of them. | Operators may believe controls exist when they do not. | Remove or clearly mark inactive options until enforcement and denial tests exist; fail closed on unsupported configured policies. |
| P1 | Persistence transition is non-atomic/incomplete | Executor writes agent result and checkpoint separately; final completed/failed status is not checkpointed after loop exit. Tables are created by handwritten SQL despite Drizzle scripts. | Resume can repeat work or restore stale `running` state; schema drift and partial writes are possible. | Use migrations and a transaction per transition; define idempotency, terminal checkpoint, resume cursor, and corruption behavior. |
| P1 | DLP wrapper corrupts result shape | `DlpFilter.redact()` returns an object, and `ToolRegistry` assigns it to `result.data` where callers expect the original output type. | Tool responses change under strict mode, potentially breaking agents while still failing to protect outbound prompts. | Make redaction typed and explicit; return cleaned data separately from audit metadata; test every egress path. |
| P1 | Sovereign mode is only model selection | `ModelRouter` selects a local model string, but telemetry defaults to localhost HTTP, MCP inherits environment, host shell may network, and no global egress gate exists. | “Sovereign” cannot be relied on as a privacy/security property. | Define sovereign as an enforceable deployment invariant and prove it with denied-network integration tests. |
| P2 | Core is coupled to concrete SQLite | `WorkflowExecutor` requires `ForgeStore`; `CcrStore` reaches through `ForgeStore.sqlite`. | Testing, alternative storage, transactions, and tenancy are harder; raw access bypasses invariants. | Introduce the smallest persistence interface justified by executor needs, not a broad repository abstraction. |
| P2 | Configuration has split vocabularies and surprising precedence | Runtime uses `RuntimeMode`; config uses `IntegrationMode`. Loader uses `.agent` and `.agent-engine`; plan uses `forge.yaml`. Global egress replaces repo egress entirely. | Operators cannot predict which settings affect execution; defaults may silently disable intended local choices. | Write an ADR and truth table for sources, precedence, environment overrides, validation, and fail-closed behavior; add table-driven tests. |
| P2 | Memory/CCR retain sensitive plaintext indefinitely | `memory_store` and `compression_cache.original_content` store plaintext; no encryption, TTL enforcement, size cap, tenant key, purge, or access policy exists. | Local secret persistence, disk exhaustion, cross-run disclosure, compliance risk. | Keep disabled for V1 or add explicit scope, encryption/keying, retention, quotas, deletion, and sensitive-content handling. |
| P2 | Hash key is truncated without collision policy | `CcrStore.hash()` stores only 16 hex characters and uses hash as primary key. Conflict updates only access time. | Rare collisions can retrieve wrong content; metadata/content are not refreshed. | Use a full digest or collision verification and namespace by workspace/thread; test collision behavior. |
| P2 | FTS maintenance is inconsistent | `ForgeStore.saveMemory()` uses `last_insert_rowid()` after an upsert and manually inserts into FTS without delete/update triggers. | Updated memories can point to wrong rows or create stale/duplicate index data. | Use external-content FTS triggers or transactional explicit rowid lookup; add update/delete/rebuild tests. |
| P2 | Observability is not auditability | OTel spans exist, but tool policy inputs/decisions, approvals, redaction metadata, actor, hashes, and outcomes are not durably recorded. | Incident reconstruction and compliance claims are unsupported. | Define a security audit schema and retention policy; keep telemetry opt-in and distinct from authoritative audit logs. |
| P2 | Supply-chain surface exceeds behavior | Many provider/UI/process packages are production dependencies but unused; normal install reported conflicting vulnerability summaries. | Larger attack surface and upgrade burden with no current value. | Inventory actual imports, remove unused runtime dependencies in a later authorized change, pin versions, add SBOM/provenance/scanning. |
| P3 | Generated/premature abstractions | Cognitive memory schema, compression taxonomy, steering, telemetry, branded IDs, and multiple config domains exist before a runnable baseline. | Maintenance load and false completeness obscure the critical path. | Freeze advanced features; stabilize one secure single-agent flow and add abstractions only from demonstrated variation. |

## Coupling and sovereignty summary

Provider-specific SDKs do not currently leak into core source imports; the dispatcher uses `LanguageModelV1` and an injected resolver. They do leak into the package boundary and lockfile, where incompatible provider generations already prevent clean installation. SQLite leaks through both workflow and compression boundaries. Policy types are disconnected from execution, so nominal separation among inference, orchestration, tools, policy, memory, persistence, evaluation, and observability is mostly directory-level rather than enforceable runtime separation. There is no evaluation subsystem.

The largest sovereignty violation is semantic: a mode name is treated as a routing preference rather than a process-wide information-flow guarantee. Host tools, MCP, provider resolution, environment inheritance, telemetry, caches, and logs all need to participate in the invariant.

## Migration risk

Do not build more features directly on current public shapes. First decide which exported interfaces are intentional. In particular, changing AI SDK generations may alter model/tool APIs; correcting DLP changes result behavior; transactional persistence changes resume semantics; and enforcing paths/commands will reject operations that currently succeed. These are desirable corrections but require explicit baseline fixtures and compatibility decisions.
